## 開発ルール（必ず守ること）

### 1. 実装フロー
```
仕様確認・更新 → ペルソナレビュー(3回) → 実装 → チェックリスト全項目確認 → デプロイ
```
- 仕様書を先に更新してから実装する
- 機能追加・変更のたびに既存仕様への影響をチェックリストで確認する
- JS構文チェック（node --check）を毎回実行する

### 2. ペルソナレビュー
4名のペルソナが指摘ゼロになるまで繰り返す（最低3回）
- 田中（モバイルUXデザイナー）
- 鈴木（情報アーキテクト）
- 山田（アクセシビリティ専門家）
- 佐藤（フィールドUXリサーチャー）

### 3. テスト項目（毎回確認）
| # | 項目 | 確認方法 |
|---|------|---------|
| T01 | JS構文エラーなし | node --check |
| T02 | 初期表示：地図は自動選択しない | コード確認 |
| T03 | 検索バーとコンパスが重ならない | CSS数値計算 |
| T04 | カテゴリフィルターが全カテゴリで動作する | filterByCatロジックテスト |
| T05 | パネル展開時に現在地ボタンが隠れない | openPanel確認 |
| T06 | 航空写真選択時のzoom自動調整 | loadMapロジック確認 |
| T07 | 住所検索が動作する（Nominatim API） | doSearch関数確認 |
| T08 | ライセンス表記（OSM・国土地理院） | HTML確認 |
| T09 | PCサイドバーとスマホパネルの両方が動作 | media query確認 |
| T10 | Supabase接続設定が正しい | URL・KEY確認 |

## アーキテクチャ

### 技術スタック
- フロントエンド: HTML/CSS/JavaScript（単一ファイル）
- 地図ライブラリ: Leaflet.js 1.9.4
- ベース地図: OpenStreetMap（常に最背面、zIndex:1）
- レイヤー: GeoJSONベクター or 地理院タイル（zIndex:2）
- BaaS: Supabase（地図メタデータ管理）
- ホスティング: GitHub Pages（immersamap.github.io）
- 住所検索: Nominatim API（OpenStreetMap）

### ファイル構成
```
immersamap.github.io/
├── index.html        # メイン（利用者向け）
├── create.html       # 地図作成者向け
├── kawagoe.jpg       # 川越城下絵図サムネイル
└── kawagoe.html      # 旧ベクターデモ（参照用）
```

### ズームレベル別の案内メッセージ（仕様）
- zoom < zoom_min → 「もっとズームインしてください（拡大するとデータが表示されます）」
- zoom > zoom_max → 「もっとズームアウトしてください」
- データなし（tileerror）→ 「この地域・年代のデータがありません」

## 機能仕様

### 住所検索
- API: Nominatim（OpenStreetMap）
- エンドポイント: https://nominatim.openstreetmap.org/search
- パラメータ: q, format=json, limit=5, accept-language=ja
- 入力後400ms debounce
- Enterキーでも検索実行
- 結果タップで地図がflyTo（zoom:16, duration:1.2s）
- エラー時: 「検索エラーが発生しました」トースト表示
- 結果なし: 「見つかりませんでした」表示

### レイヤー表示
- OSMが常にzIndex:1（ベース、非表示にしない）
- オーバーレイはzIndex:2
- 透明度: 0〜100%（デフォルト80%）
- レイヤーON/OFF: ベクター地図のみ（ピル形式）

### GPS・現在地
- 現在地ボタン（青い丸アイコン）タップでGPS取得
- 取得成功: 青い丸マーカー＋パルスアニメ、zoom:16にflyTo
- 取得失敗: 「GPS取得失敗」トースト
- 継続追跡: watchPositionで位置更新

### コンパス
- タップで北向きリセット
- デバイス方位センサー対応（iOS: 権限リクエスト）

## 既知の制限事項

1. Claudeサーバーからimmersamap.github.ioへの直接アクセス不可
   → コードレベルのテスト + ユーザーによるスクリーンショット確認で対応
2. Claudeサーバーから地理院タイルへのアクセス不可（403）
   → ブラウザからは正常取得可能
3. 年度別空中写真（nendophoto）はzoom14以上必要
4. 1936年・1928年の空中写真は特定地域のみ
